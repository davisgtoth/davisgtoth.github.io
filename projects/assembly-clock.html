Project Overview

This project began with a practical need for a desk clock and a Terasic DE0-CV FPGA board that was simply gathering dust. I used this as an opportunity to dive into bare-metal engineering, transforming the board into a fully functional digital clock entirely in 8051 Assembly. This approach allowed me to manually interface with the board's 7-segment displays and buttons, building a responsive, interactive system from the ground up. The full source code is available on my Github!

The core challenge lay in shifting from high-level software to low-level hardware constraints. Unlike coding in Python, this environment required manual memory management, direct register manipulation, and strict adherence to a limited instruction set. With no native if/else structures available, I relied on status flags and conditional jumps to orchestrate complex logic. Ultimately, transforming raw binary data into a precise HH:MM:SS timekeeper and calendar was a rewarding challenge that forced me to problem-solve in an entirely new way.

Display Modes

To elevate this project beyond a trivial counter, I focused on designing a genuinely usable interface. I programmed the board's four push buttons to manipulate specific status flags in memory, enabling seamless toggling between distinct display modes. By default, the system operates in a 24-hour format but can be instantly switched to a 12-hour standard. A separate control triggers a temporary "Date View" that displays the current date (MM/DD/YY) for two seconds before automatically reverting to the clock.

Since this device lived on my desk, ergonomics were a priority. To eliminate the distraction of constantly flickering seconds in my peripheral vision, I implemented a mode to hide them — swapping the digits for AM/PM indicators when in 12-hour format. I also added a "Sleep Mode" to blank the bright 7-segment displays entirely. Crucially, the underlying timekeeping logic continues to run in the background regardless of the visual state, ensuring the clock maintains perfect accuracy.

Setting the Clock

To ensure practical utility during events like Daylight Savings or power outages, I engineered a robust manual configuration system. Toggling specific board switches transitions the processor into a "Set State", allowing the user to directly modify the time or date. In this mode, the active digit pair (seconds/minutes/hours) is visually highlighted by a blinking effect. The push buttons serve as a navigation interface: two buttons shift the selection left or right, while the other two independently increment or decrement the value.

The true complexity lies in the backend validation logic. I implemented custom boundary checks that dynamically adjust based on the context. For example, when setting the date, the system automatically caps the day at 30 for June or 31 for January. The logic even accounts for leap years, permitting February 29th only when appropriate. All inputs feature "roll-over" logic, ensuring that incrementing past a maximum value seamlessly loops back to the start, providing a smooth and intuitive user experience.
